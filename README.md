DESCRIPTION
I wanted to make a game as final project and the idea came from cs50 final project webpage where one of the ideas was 'a game created using lua with LOVE'. The implementation was fairly simple compared to writing the code, as certain parts require long time to write but overall relatively doable.

This project is build using mainly lua files to write code and LOVE2D where the game is run on. Additionally sounds and graphics also involved in this project which will be explain later. This is basically a puzzle type game where users control movements of a robot using WASD keys and rotating map using left and right arrows to collect wrenches each level as increasing difficulty when progressing levels. The game consist of 6 stages with 6 bonus stages that can be unlocked once all stages are completed. This game consist of many features such as rotating camera angle, shader effects and lighting, bonus levels. Furthermore it consist of teleporters, ice blocks and trampoline which helps the certain levels to be more fun to solve.

The project consists of 10 lua files and 3 additional folders. The intro.lua file is basically the introduction of the game. This file deals with the transitioning, timer for example. When the introduction screen is loading, it contain the time for the intro to load and how long it shows up before moving on. The screen background is set to black while the timer counts until 3.8 seconds is reached which is the introtime. Once its done , the lobby page will appear which will be explain later. Values.lua file sets the respawn on character on the map. It also contain the bonus level and how it can be unlocked. In addition , it also deals with block size, rotation and player's fall , jump and move speed. Menu.lua initializes the dialogue before each level and mainly the transition. It Main.lua. This file serves as the central foundation for the game, initializing settings, loading global media assets, and driving the core execution lifecycle. It acts as the backbone that connects independent states like the menu, intro, gameplay, and credits. An example is the screen scaling system. takes a small, retro game resolution and scales it up cleanly to fill your modern computer screen without blurring the pixel art. Game.lua has many functions inside. The file manages the core systems of a pseudo-3D, grid-based puzzle game built with the LÖVE framework. By analyzing three-dimensional grid coordinates, it projects isometric layouts onto a flat 2D screen while continuously calculating game physics and real-time tile updates. It handles frame-by-frame player logic, gravity falls, and camera viewpoints based on your movement. The game also features complex level mechanics where the layout of the map can completely shift around you. An example of a game mechanic is the ice block.It works like an automated conveyor belt. Normally, when you press a directional arrow key, you move exactly one step and stop. However, when the script detects that your character has stepped onto an ice grid coordinate, it temporarily steals your controls and locks your direction. It then automatically pushes your character across the adjacent grid spaces line-by-line until you hit a solid wall or a dry tile, which safely restores your manual controls. Class.lua allows creating a template and then create many subcategories of it. Conf.lua or configuration sets the name of game, the width and height and uses icon image. Not only that, it disables controller support to save memory and turns off built-in physics engine. Ending.lua is when the player completes all the level. A screen will show indicating that the player has finish the game and a message says congratulations britebot has retrieved all the wrenches, thank you for playing!" Shaders.lua is self-explanatory. It is use for better quality gameplay which can be changed in settings however the game might run slower than normal as it takes time to load in better graphics. Buttons.lua contains buttons which can be seen in lobby like settings, quit, start and help etc. which can be click. Graphics folder contain images like wrenches, bot, blocks and background images which can be seen throughout levels. Additionally , lock.png can be seen for locked levels. Lastly, it also have the logo.png and menu arts which can be see in lobby along with buttons mentioned earlier. Level files contain level 1 to level where users can play. There is also 6 more bonus level set on nighttime but it requires users to complete a certain action for it to appear and be played. Sound files contains sounds when user collect wrench, push block, jump on trampoline etc.
